Database Per Service

Each service owns its data. No shared tables or database-level foreign keys across services to ensure maximum scalability.

Logical Joins

Joins are performed at the API level. We store external IDs (e.g. courseId) and resolve details through inter-service REST calls.

Messaging Integrity

RabbitMQ ensures that critical state changes (like payments) are propagated across services reliably and asynchronously.

Authoring Flow Instructor Content Creation & Publishing
sequenceDiagram participant I as Instructor participant G as API Gateway participant C as Course Service participant L as Lesson Service participant N as Notification Service I->>G: POST /api/course (Draft) G->>C: Create Course Entity C-->>I: 201 Created (courseId) I->>G: POST /api/lesson (Add Video) G->>L: Save Lesson metadata L-->>I: Success I->>G: PUT /api/course/publish/{id} G->>C: Update status to PUBLISHED C->>N: Trigger Announcement N-->>I: Dashboard Update
Event Flow Secure Checkout & Enrollment Trigger
sequenceDiagram participant U as Student (UI) participant G as API Gateway participant P as Payment Service participant R as Razorpay API participant MQ as RabbitMQ (Bus) participant E as Enrollment Service U->>G: POST /api/payments/create-order G->>P: Forward Request P->>R: Generate Order ID R-->>P: order_id P-->>U: Return order_id U->>R: Open Checkout Modal (JS SDK) R-->>U: payment_id & signature U->>G: POST /api/payments/verify G->>P: Verify Signature P->>MQ: Publish event [payment.success] MQ->>E: Consume [payment.success] E->>E: Create Enrollment Record E-->>U: WebSocket: "Course Unlocked!"
